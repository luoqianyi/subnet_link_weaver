## ⚠️ 契约 wiring 验证提醒

本评审仅验证了**设计文档**和**需求分析**的内部一致性（如字段定义、方法签名、错误类型）。
**未验证代码是否真正实现了设计中定义的每个类/方法/异常**。

以下契约项需在代码实现阶段进行机械验证：

1. 设计文档中定义的每个类 → 代码中是否有对应的 `class` 定义
2. 设计文档中定义的每个方法 → 代码中是否有对应的方法实现
3. 设计文档中定义的每个异常 → 代码中是否有对应的异常类
4. 设计文档中定义的每个数据模型 → 代码中是否有对应的 dataclass

**由开发阶段保证覆盖率 ≥95%，否则不得发布。**
