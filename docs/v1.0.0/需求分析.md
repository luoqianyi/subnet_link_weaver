# Subnet Link Weaver v1.0.0 需求分析

## 1. 项目背景

### 1.1 问题描述
在局域网环境中，多台电脑需要互相访问进行联调测试。默认情况下，路由器分配的 IP 地址可能不在同一网段，或者需要手动配置静态 IP，过程繁琐且容易出错。

### 1.2 解决方案
开发一个 Windows 桌面应用程序，帮助用户一键配置多台电脑的 IP 地址，使它们能够在同一个自定义网段（如 192.168.88.X）中互相访问。

## 2. 功能需求

### 2.1 核心功能

#### F1: 网络适配器检测
- **描述**: 自动检测当前系统的所有网络适配器
- **输入**: 无
- **输出**: 网络适配器列表（名称、状态、当前 IP、网关、DNS）
- **优先级**: P0

#### F2: 静态 IP 配置
- **描述**: 将动态分配的 IP 转换为静态 IP
- **输入**: 网络适配器名称、IP 地址、子网掩码、默认网关、DNS
- **输出**: 配置结果
- **优先级**: P0

#### F3: 多 IP 挂载
- **描述**: 在同一网卡上添加额外的 IP 地址
- **输入**: 网络适配器名称、要添加的 IP 地址、子网掩码
- **输出**: 配置结果
- **优先级**: P0

#### F4: 防火墙管理
- **描述**: 一键关闭/开启 Windows Defender 阁火墙
- **输入**: 开关状态
- **输出**: 操作结果
- **优先级**: P0

#### F5: 连通性测试
- **描述**: 使用 ping 命令测试目标 IP 的连通性
- **输入**: 目标 IP 地址
- **输出**: ping 结果（成功/失败、延迟）
- **优先级**: P0

### 2.2 辅助功能

#### F6: 配置保存/加载
- **描述**: 将当前配置保存到文件，支持加载已有配置
- **输入**: 配置文件路径
- **输出**: 配置内容
- **优先级**: P1

#### F7: 批量配置
- **描述**: 同时配置多台电脑（需要在每台电脑上运行程序）
- **输入**: 目标 IP 列表
- **输出**: 批量操作结果
- **优先级**: P2

#### F8: 配置模板
- **描述**: 预设常用配置模板（如 192.168.88.X 网段）
- **输入**: 模板名称
- **输出**: 预填的配置值
- **优先级**: P2

## 3. 非功能需求

### 3.1 系统要求
- **操作系统**: Windows 10/11 (64-bit)
- **权限**: 需要管理员权限运行
- **依赖**: Python 3.10+, PyInstaller

### 3.2 性能要求
- 网络适配器检测时间 < 2 秒
- IP 配置操作时间 < 5 秒
- 防火墙操作时间 < 3 秒
- ping 测试时间 < 10 秒

### 3.3 安全要求
- 所有网络操作需要管理员权限
- 配置文件不包含敏感信息
- 操作日志记录所有配置变更

### 3.4 可用性要求
- GUI 界面直观易用
- 操作步骤不超过 5 步
- 提供操作向导模式
- 错误信息清晰明确

## 4. 用户故事

### 4.1 主要用户
- 开发人员（需要多台电脑联调）
- 测试人员（需要模拟多机环境）
- 运维人员（需要配置网络环境）

### 4.2 用户故事列表

#### US1: 检测网络适配器
**作为** 开发人员
**我希望** 能够看到当前电脑的所有网络适配器及其配置
**以便** 选择要配置的适配器

#### US2: 配置静态 IP
**作为** 开发人员
**我希望** 能够将动态 IP 转换为静态 IP
**以便** 固定当前网络配置

#### US3: 添加额外 IP
**作为** 开发人员
**我希望** 能够在同一网卡上添加额外的 IP 地址
**以便** 使电脑能够在多个网段中通信

#### US4: 关闭防火墙
**作为** 开发人员
**我希望** 能够一键关闭 Windows 阁火墙
**以便** 允许跨网段的网络通信

#### US5: 测试连通性
**作为** 开发人员
**我希望** 能够测试与其他电脑的连通性
**以便** 确认配置是否成功

## 5. 约束条件

### 5.1 技术约束
- 仅支持 Windows 操作系统
- 使用 `netsh` 命令进行网络配置
- 使用 PowerShell 命令管理防火墙
- 使用 `ping` 命令测试连通性

### 5.2 业务约束
- 需要管理员权限运行
- 配置变更需要用户确认
- 操作需要记录日志

### 5.3 环境约束
- 需要在同一局域网内
- 目标 IP 不能与其他设备冲突
- 防火墙关闭后需要提醒用户恢复
